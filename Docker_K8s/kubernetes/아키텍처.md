# Workload
## Deployment, ReplicaSet
- 상태 없는 파드 실행
- Spring API Server 같은 애플리케이션 파드에 적합
- ReplicaSet은 지정한 수의 pod를 유지
- Deployment는 ReplicaSet에 배포 운영 기능을 추가한 오브젝트
    - pod의 image가 변경될 떄 ReplicaSet은 기존 pod를 유지한채 새 pod를 생성하지만 Deployment는 새 pod 생성, 기존 pod 삭제 작업을 진행
    - 변경되어도(Pod는 삭제되어도) 기존 ReplicaSet의 기록은 남긴다. 이를 이용해 롤백을 지원한다

```yml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3 # ReplicaSet
  selector:  # ReplicaSet이 Pod 찾는 방법
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

```

## DaemonSet
- 노드에 반드시 1개 실행되야하는 Pod를 실행
- 노드가 추가 되면 자동으로 Pod생성, 삭제되면 자동으로 pod 삭제
- 로그 수집기, 모니터링 같은 애플리케이션 파드에 적합

## StatefulSet
- 상태 있는 Pod 실행
- pod 이름 고정, 순서 보장, Persistent Volume 이용
- 데이터베이스, 분산시스템(Kafka, ElasticSearch) 애플리케이션 파드에 적합

## Job
- 한번 완료될 때까지 실행하고 끝내는 작업 정의
- Pod가 죽으면 다시 켤 시 재시도, 작업 완료되면 더 이상 작업을 실행하지 않는다

## CronJob
- 정해진 시간마다 실행
- ConcurrentPolicy로 실행 시간이 되어도 이전 작업 완료 상황을 보고 대기 시킬 수 있다

# Network
- 실무에서는 Ingress와 ClusterIP Service 조합을 많이 사용

## Service
- Network-L4
- Pod들을 묶어 고정된 IP를 제공
    - Pod는 죽고 재실행될 떄마다 IP가 바뀐다
- ClusterIP
    - 클러스터 내부안 pod끼리 접근 허용 
- NodePort
    - 클러스터 외부 접근 허용
    - 디버깅용
- LoadBalancer
    - NodePort 기반 외부 CSP에서 생성한 LoadBlancer 연결

## Ingress
- 사전 의미: 입장
- ingress가 지정한 host/path로 온 요청을 연결된 ClusterIP service로 전달
- HTTP/HTTPS 요청 도메인 기반 라우팅 입구
- Network-L7
- 하나의 Ingress로 여러 Service 관리하기 위해 사용
- HTTPS의 SSL을 종료해 컨테이너 내부에서는 HTTP 요청 처리할 수 있도록 변경

## Ingress Controller
- ingress는 전달 규칙일 뿐 실제 전달 하는 주체는 IngressController가 한다
- nginx, kong, gke 같은 ingress controller를 설치해 이용
- ingress controller 용 service(LoadBalancer)도 필요하다

## service
- pod에 쿠버네티스 외부에서 접근할 수 있는 고정 네트워크 주소 부여
- pod가 죽어서 재시작해도 외부에서 동일 네트워크로 접근할 수 있도록 한다
- 종류
    - NodePort(테스트 용도), ClusterIP(내부 통신용), LoadBalancer, ExternalName(외부 DNS와 연결)
- selector 없이도 매니패스트 정의 가능

    - endpoint를 지정
- network-L4 지원

## ingress
- 외부 요청을 클러스터 내부 ClusterIP service로 라우팅 하는 규칙
- HTTP/HTTPS 기반
- 이름 기반 가상 호스팅
    - 하나의 IP를 여러 dns로 제공
- ingress 내부에서 https 로 오는 요청의 ssl를 해제하고 pod안에서는 http 요청을 처리하게 할 수 있다
- 프록시 설정
    - ingress는 동일 지점으로 온 요청을 경로에 맞춰 전달하는 역할


## ingress controller
- ingress의 규칙을 실체 처리

## namespace
- 클러스터 내 자원 구분 단위
- dev, stage, prod 처럼 하나의 쿠버네티스 클러스터의 자원을 나눠 관리
- pod, service, deployment, replicaset, configmap, secert 들이 namespace로 구분해서 관리

- 전역 자원
    - 네임스페이스 안에가 아니라 클러스터에서 공유하는 자원
    - node, PersistenVolume, Namespaces, ClusterRole 들이 전역 자원 관리
