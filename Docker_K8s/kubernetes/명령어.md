## 문법
- apiVersion  
  - Resource 마다 정해져 있다
  - v1: core api
    - Pod, Service, ConfigMap, Secret, PVC
  - apps/v1
    - Deployment, StatefulSet, DaemonSet, ReplicaSet
  - batch/v1
    - Job, CronJob
  - networking.k8s.io/v1
    - Ingress

```yml
apiVersion:  # api group/version
kind: # 만들고 싶은 리소스
metadata: # 리소스 정보
    name: # 이름
    labales: # 리소스 그룹 이름
      app:
    namespace: # 리소스 격리
spec: # 내가 원하는 리소스 상태 선언
status: # 현재 상태
```

## Deployment
```yml
spec:
  replicas: 3 # pod 숫자
  
  selector:
    matchLabels:
      app: # pod 찾는 기준
  
  template: # Pod 상태 정의
    metadata:
      labels:
        app:
    spec: 
      containers: # 컨테이너 정의 
        - name:
          image:
          ports:
            - 
          env:
            - name:
              value:
          valueForm: # ConfigMap/Secret 연결
            configMapKeyRef:
              name:
              key:
          volumeMounts: # Volume 마운트
            - name:
              mountPath:
      volumes: Volume 사용
        - name:
```

# Igress
```yml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: api-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  rules:
    - host: api.myservice.com # 해당 host로 오는 요청에 대해 ingress가 담당
      http:
        paths:
          - path: /users # 해당 path로 오는 요청은 어떤 service로 전달할지 지정
            pathType: Prefix
            backend:
              serviceName: user-service # https://api.myservice.com/users → user-service:80 로 전달
              serverPort: 80

```

- ALB(Aws Load Balancer)는 ingress yml 파일을 읽어 Ingress Controller를 만든다
- Route53(AWS DNS)에서 url -> ALB DNS를 등록
