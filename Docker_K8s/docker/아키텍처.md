# Docker
- 켄터이너 기술(호스트 OS에서 독립된 애플리케이션 실행을 환경을 제공하는 기술) 소프트웨어
- 해결하고 싶은 문제: 한번 빌드하면 어디서나 동일하게 실행 가능
- 사용 프로세스
    - build: 이미지 작성
    - ship: 이미지 배포및 공유
    - run: 컨테이너 실행
- VM이랑 차이   
    - VM은 하드웨어 수준에서 가상화, 완전한 컴퓨터 처럼 동작 
    - Docker의 컨테이너는 OS 수준에서 가상화, host os 커널을 공유

## image
- 컨테이너를 만드는 설계도

## container
- 이미지 기반으로 생성한 인스턴스
- 애플리케이션 실행 환경

## docker daemon(engine)
- 컨테이너를 생성, 실행하는 프로세스

## docker cli
- 도커 관련 명령어 인터페이스

## Build
- docker build 이미지이름 context

### DockerFile
- 이미지 생성 절차 기록 파일
- 베이스 이미지는 DockerHub에서 다운받아서 지정 이미지에 포함된다
- 하나의 이미지는 파일 시스템 기반 여러 계층 폴더/파일로 이루어저 있다
- 도커 파일의 각 명령어 결과를 레이어처럼 쌓여서 한번 build시 결과물 레이어들을 캐싱해 다음 빌드시 변경 사항부터 시작한다
- 레이어 캐시는 Docker Engine의 /var/lib/docker 폴더에 저장된다

```
FROM  // 베이스 이미지 선택, 멀티 스테이지 빌드 단위
WORKDIR // 작업 폴더 위치
RUN // 실행 명령어들 정의
```

### Context
- 컨테이너 빌드시 참고하는 폴더 범위

## Run
### docker-compose.yml
- 여러 컨테이너들 실행을 묶은 설정 파일
- 네트워크 설정, 환경 변수 설정, 볼륨 설정
