# Consumer
- 카프카에서 브로커의 메세지를 읽어 처리하는 애플리케이션
- 하나의 컨슈머가 여러 파티션을 읽을 수 있다
- 컨슈머는 브로커로 부터 메세지를 복사한 후 당겨(poll) 읽는다(commit)
- (consumer group, topic, partition) -> offset

# Consumer Group
- 하나 이상 컨슈머를 논리적으로 묶은 단위
- 목적: 부하 분산
- 하나의 파티션은 같은 컨슈머 그룹안 컨슈머 중 하나만 읽을 수 있다
- 같은 파티션이더라도 오프셋은 각 그룹 단위로 관리 된다 
- 컨슈머 그룹을 설정하지 않은 컨슈는 독립된 컨슈머 그룹으로 친다
